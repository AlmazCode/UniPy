Эдитор движка, представляет собой простой в использовании инструмент, который обладает несколькими важными функциями. Перед началом работы с ним рекомендуется внимательно изучить документацию, особенно разделы, описывающие object_docum и text_docum.

Один из основных инструментов, доступных в эдиторе - это специальные функции, которые можно использовать для инпутов компонентов. Например, для компонента "x" можно прописать функцию centerx(), которая вернет центр позиции "x" объекта на экране. Вот список всех возможных функций для инпутов:

    centerx() - возвращает центр оси "x".
    centery() - возвращает центр оси "y".
    top() - возвращает координату верхней точки экрана по оси "y".
    left() - возвращает координату левой точки экрана по оси "x".
    bottom() - возвращает координату нижней точки экрана по оси "y".
    right() - возвращает координату правой точки экрана по оси "x".
    ww() - возвращает ширину экрана.
    wh() - возвращает высоту экрана.
    
Использование этих функций делает работу с инпутами компонентов более удобной и эффективной.

АССЕТЫ:

Для того чтобы импортировать ассеты в ваш проект, вам нужно воспользоваться специальной функцией, доступной в эдиторе движка. Для этого нажмите на четвертую кнопку справа, и откроется проводник движка, где вы сможете увидеть все ассеты в вашем проекте. Затем, нажмите на кнопку "+", чтобы открыть проводник на вашем телефоне и выбрать файл, который вы хотите импортировать в ассеты.

Важно отметить, что в ассетах вы можете также переименовывать файлы и создавать новые папки. Для переименования файла выберите его в списке, и нажмите на кнопку "Переименовать", которая появится выше списка файлов. Для создания новой папки в ассетах воспользуйтесь четвертой кнопкой слева, которая позволит создать новую папку в текущей директории.

Таким образом, вы можете легко импортировать и управлять ассетами в вашем проекте, используя удобный интерфейс эдитора движка. Это позволит вам сэкономить время и сделать вашу работу более эффективной.

СКРИПТЫ:

В UniPy, расположение скриптов имеет особое значение и должно соответствовать определенным правилам. Все скрипты должны иметь расширение .py и храниться в корневой папке проекта.

Важно отметить, что скрипты нельзя располагать в других папках проекта, даже если они находятся внутри корневой папки. Например, если в проекте "game" вы создадите папку "scripts" и поместите все скрипты в нее, то движок не сможет найти их.

Правильное расположение скриптов в UniPy:

```
.projects/game/script.py
```

Неправильное расположение скриптов в UniPy:

```
.projects/game/scripts/script.py
```

Также, так как UniPy OpenSource инструмент, то в файле settings.py есть уже готовые настройки UI движка, которые вы можете менять как захотите.

Соблюдение правил расположения скриптов в UniPy поможет избежать ошибок при работе с движком и обеспечит более эффективное использование ресурсов.
